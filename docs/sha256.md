# Documentation — SHA-256 Final

---

## Rôle

`sha256` applique le padding final et produit le digest hexadécimal
de 64 caractères (256 bits).

---

## Pipeline

```
sha256(ctx)
    ↓
sha256_padding(ctx)     → complétion + transform final
    ↓
ctx->h[0..7]            → 8 mots uint32_t = 256 bits
    ↓
word_to_hex × 8         → 64 chars hex
    ↓
char *digest[65]        → retourné à l'appelant
```

---

## Implémentation

```c
static void     word_to_hex(uint32_t word, char *buf)
{
    char    hex[] = "0123456789abcdef";
    int     i;

    i = 7;
    while (i >= 0)
    {
        buf[i] = hex[word & 0xF];
        word >>= 4;
        i--;
    }
}

char    *sha256(t_sha256_ctx *ctx)
{
    char    *digest;
    int     i;

    if (!ctx)
        return (NULL);
    sha256_padding(ctx);
    digest = ft_calloc(1, 65);
    if (!digest)
        return (NULL);
    i = 0;
    while (i < 8)
    {
        word_to_hex(ctx->h[i], digest + (i * 8));
        i++;
    }
    digest[64] = '\0';
    return (digest);
}
```

---

## `word_to_hex` — conversion big-endian

SHA-256 affiche les mots en **big-endian** — l'octet le plus significatif
en premier, comme OpenSSL.

```c
// i = 7..0 → remplit de droite à gauche
// chaque itération extrait le nibble le moins significatif
// et recule l'index → le MSB finit à gauche

word = 0xba7816bf

i=7 : buf[7] = hex[0xf] = 'f'
i=6 : buf[6] = hex[0xb] = 'b'
...
i=0 : buf[0] = hex[0xb] = 'b'

→ "ba7816bf" ✓
```

---

## Responsabilité mémoire

```c
digest = ft_calloc(1, 65);   // alloué ici
// ...
return (digest);             // l'appelant doit free()
```

L'appelant (`compute_sha256`) est responsable de libérer le digest :

```c
hash = sha256(ctx);
// utiliser hash
free(hash);
```

## Comparaison avec MD5

| | MD5 | SHA-256 |
|---|---|---|
| Taille digest | 32 chars (128 bits) | 64 chars (256 bits) |
| Mots | 4 × uint32 | 8 × uint32 |
| Fonction hex | word_to_hex (LE) | word_to_hex (BE) |
| Padding | dans md5() | dans sha256() |
